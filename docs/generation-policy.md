# CollabVaultX Generation Policy

This policy defines what Codex should generate from a validated Research
Package and what should remain manually reviewed.

## Source Of Truth

The Research Package is the pre-import source of truth. `data/*.json` is the
normalized site model after validation. Generated HTML should be reproducible
from the normalized data plus templates.

Codex should not invent new campaign facts while generating pages. If the
package lacks evidence, Codex should fail validation or record an unresolved
question.

## Generate First

These pieces are stable and should move toward full generation first:

- campaign product grid
- parent card image and title links
- item page shell
- metadata, canonical URLs, OGP, and Twitter metadata
- breadcrumbs
- hero media
- Marketplace Finder
- related item navigation
- bottom back links
- sitemap entries
- generated page markers and regeneration checks

These areas already have renderer/checker coverage or clear data contracts.

## Generate Next

These can become generated after the first pilot proves the package quality:

- campaign hero copy
- campaign overview cards
- item overview cards
- product detail sections
- official source/reference sections
- structured archive notes based on evidence records

The generator should still preserve a small reviewed override area for unusual
campaign caveats.

## Keep Manual Or Human-Reviewed

Do not fully automate these decisions yet:

- official source meaning when wording is ambiguous
- rights, endorsement, and asset-use risk
- affiliate disclosure readiness
- source conflict resolution
- item grouping when official pages split or combine goods inconsistently
- campaign-specific legal or participation caveats
- live marketplace price, stock, or authenticity claims

## Page Volume Rules

Large-scale generation must avoid thin pages:

- Generate item pages only for `pageDecision.decision === "generate"`.
- Do not generate character-variant pages by default.
- Use `parent-card-only` aggressively for simple single-design goods.
- Use `defer` when source, asset, or search-value evidence is not strong enough.
- Require unique `summaryEn` for generated pages.
- Require a source-backed reason for every generated item page.

## File Ownership

Generated files should be marked in future generator work so manual edits do not
quietly drift:

```html
<!-- Generated by CollabVaultX. Edit the Research Package or data JSON. -->
```

Until full-page generation lands, Codex should keep using renderer/checker
scripts for shared fragments and avoid hand-edited repeated HTML.

## Monetization Constraints

Marketplace links are reference searches unless affiliate support is explicitly
enabled. Affiliate links require:

- `isAffiliate: true`
- a real affiliate URL
- `rel` containing `sponsored`
- `disclosureRequired: true`
- visible page disclosure

The default state is non-affiliate reference search links with
`nofollow noopener noreferrer`.
